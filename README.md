# Cloud Config Right Sizing Agent

A low-code agent that parses customer SOW (Statement of Work) documents and generates chamber configuration JSON for CICD deployment pipelines.

**Primary path:** Gemini Enterprise Agent Designer
**Supporting tools:** Local Python/Gradio app for sanitization, validation, and scoring

---

## Part 1: Sanitize SOW (Privacy Layer)

Extracts text from a SOW PDF and redacts customer PII before any content is processed by the model. Use this to prepare sanitized input for Agent Designer.

| File | Purpose |
|---|---|
| `extract_pdf.py` | Extracts text from PDF files (PyMuPDF primary, pypdf fallback) |
| `sanitizer.py` | Redacts emails, phones, addresses, account IDs, client names, pricing |

**What it does:** Raw SOW in → sanitized text out (PII replaced with `[REDACTED]` tags, infrastructure requirements preserved)

---

## Part 2: Validate & Score (Output Quality)

Takes the JSON config generated by Agent Designer (or any source), validates it against the CICD schema, and scores accuracy against known-good configs.

| File | Purpose |
|---|---|
| `validate_output.py` | Checks JSON against `config_schema.json` (required fields, data types, enums) |
| `score_accuracy.py` | Compares generated config to a known-good config, field by field |
| `config_schema.json` | The target JSON schema for chamber configs |
| `data/eval_configs/` | Known-good config JSONs for accuracy scoring |

**What it does:** JSON config in → validation status + field-level accuracy report out

---

## Part 3: Full Pipeline (End-to-End with Gemini API)

Runs everything automatically — extract, sanitize, generate via Gemini API, validate, and score. This is a backup/demo option if we want to show the full workflow without Agent Designer.

| File | Purpose |
|---|---|
| `call_gemini.py` | Calls Gemini (free API key, Vertex AI, or mock fallback) |
| `build_prompt.py` | Builds few-shot prompt with reference examples + schema |
| `data/reference_sows/` | Training example SOW text files |
| `data/reference_configs/` | Matching known-good config JSONs for few-shot examples |

**What it does:** SOW PDF in → sanitized text → Gemini generates config → validated + scored → all in one step

## Quick Start

```bash
git clone <your-fork-url>
cd cadence-cloud-config
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python app.py
# Open http://localhost:7860
```

## GCP Requirements (if using Part 3 with Vertex AI)

| Requirement | Details |
|---|---|
| APIs to enable | `aiplatform.googleapis.com`, `storage.googleapis.com` |
| IAM roles | Vertex AI User (`roles/aiplatform.user`), Storage Object Viewer (`roles/storage.objectViewer`) |
| Auth | `gcloud auth application-default login` or service account |

For Part 3 without Vertex AI, a free Gemini API key from [aistudio.google.com](https://aistudio.google.com/apikey) works with no GCP setup.

Parts 1 and 2 require no GCP access or API keys.
