What the Agent Extracts vs. What Gets Redacted

Extracted (kept)
  • Platform provider and region
  • Hardware specs (instance type, vCPUs, RAM)
  • Storage (backup GB, scratch GB)
  • Software licenses (product + quantity)
  • User counts and locations
  • PDK requirements
Redacted (removed)
  • SOW headings and order numbers
  • Customer/billing addresses
  • Contact info, emails, signatures
  • Terms and conditions
  • Payment schedules and pricing
  • Footer text and document IDs

PROBLEM

Manual SOW parsing is slow, inconsistent, and error-prone. Missed fields lead to deployment rework.

 DEMO

[Record 2-min demo on dev day]
APPROACH

Low-code agent in Gemini Enterprise Agent Designer + Gradio tooling for PII redaction and schema validation.

 VALUE / IMPACT
  • SOW-to-config turnaround: hours to minutes
  • Eliminates manual transcription errors
  • Privacy-first: PII redacted before model input
  • Validated, standardized output reduces rework

Production path: Automated PII preprocessing, SOWs and configs stored in Cloud Storage, validation pipeline as a quality gate, approved configs feed into Terraform for chamber provisioning.
