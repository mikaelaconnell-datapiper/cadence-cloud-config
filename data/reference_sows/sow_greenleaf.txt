STATEMENT OF WORK
Client: GreenLeaf Health Systems
Contract Date: 2026-01-10
Project: Patient Portal Cloud Migration

1. SCOPE OF WORK
GreenLeaf Health Systems requires a HIPAA-compliant cloud environment for their
patient portal application. The portal handles sensitive healthcare data including
patient records, appointment scheduling, and billing information.
The system serves approximately 1000 concurrent users including patients and staff.

2. TECHNICAL REQUIREMENTS
- Compute: The application requires robust compute for handling concurrent sessions.
  Each application server needs 8 vCPUs and 32GB RAM.
  Six application servers are required to handle peak load with redundancy.
- Storage: Total storage requirement is 5TB for patient records and application data.
  SSD storage is mandatory for database performance. IOPS requirement is 10000.
- Networking: VPC with 4 subnets (web tier, app tier, database tier, management).
  External load balancer required for patient-facing traffic.
  VPC CIDR: 10.10.0.0/16
- Services Required:
  * Cloud SQL (MySQL) - primary database
  * Cloud Storage - document and image storage
  * Cloud Monitoring - infrastructure and application monitoring
  * Cloud Logging - audit logging (HIPAA requirement)
  * Cloud KMS - encryption key management
  * Cloud Armor - WAF and DDoS protection

3. AVAILABILITY REQUIREMENTS
- High availability is mandatory (healthcare SLA: 99.99%)
- Hourly backups with 90-day retention

4. USER ESTIMATE
- 1000 concurrent users
- Expected to grow to 3000 within 12 months
